Modelo de entidad-relación

El modelo de entidad-relación es un enfoque conceptual y gráfico utilizado en el diseño de bases de datos para representar la estructura y las interrelaciones entre diferentes entidades. Propuesto por Peter Chen en la década de 1970, el modelo entidad-relación se ha convertido en un estándar en la ingeniería de bases de datos y es ampliamente utilizado para visualizar y diseñar sistemas de información.

    Claves del Modelo entidad-relación:


  1. Entidades: Las entidades son objetos o conceptos del mundo real representados en la base de datos. Cada entidad se representa mediante un rectángulo en el diagrama entidad-relación.

  2. Atributos: Los atributos son características o propiedades que describen una entidad. Cada entidad tiene atributos que capturan información específica sobre ella.

  3. Relaciones: Las relaciones modelan cómo las entidades interactúan entre sí y se representan mediante líneas que conectan las entidades asociadas.

  4. Clave Primaria: Cada entidad tiene una clave primaria, que es un atributo que identifica de manera única a cada instancia de la entidad.

  5. Cardinalidad: La cardinalidad de una relación especifica cuántas instancias de una entidad pueden estar relacionadas con una instancia de otra entidad. Puede ser uno a uno, uno a muchos o muchos a muchos, definiendo las restricciones de la relación.
